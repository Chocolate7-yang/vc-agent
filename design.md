# VC 信息聚合系统设计文档

目标是把平台信息变成投资人“5 分钟可读”的每日简报。

## 1. 架构设计

系统可以理解成 6 个环节，按流水线协作：

1. 采集环节：去 Twitter、YouTube、公众号拿新内容。  
2. 整理环节：把不同平台的字段统一（标题、作者、时间、链接等）。  
3. 过滤环节：去重、去广告、筛掉无价值内容。  
4. 解读环节：把内容改写成“结论 + 投资影响 + 风险”。  
5. 简报环节：按重要程度排版，生成日报并推送。  
6. 学习环节：根据用户反馈调整后续推荐。

一句话理解：采集环节先收全信息，过滤环节去掉噪音，解读环节讲清重点，学习环节让系统越用越懂你。

## 2. 数据采集方案

### YouTube
- **当前状态**：已实现（当前唯一上线信息源）。  
- **怎么采集**：采集环节先用频道 RSS 拿新视频，再按需调用 API。  
- **技术限制**：API 配额有限，字幕/转录不一定拿得到。  
- **应对方案**：先低成本抓标题和简介；只对高价值视频补抓字幕；时间窗口控制在近 72 小时。

### Twitter
- **当前状态**：暂未实现（规划中）。  
- **计划方案**：后续由采集环节对接官方 API，按“优先账号 + 行业关键词”抓取。  
- **主要限制**：调用次数限制、接口权限分层。  
- **计划应对**：增量抓取 + 分层频率；限流时仅保留核心账号近 24 小时内容。

### 公众号
- **当前状态**：暂未实现（规划中）。  
- **计划方案**：仅接入合规授权数据源，由采集环节的可插拔 Connector 对接。  
- **主要限制**：反爬严格、页面结构常变、合规要求高。  
- **计划应对**：优先授权源；不稳定来源降级为“标题监控”。

## 3. “高质量”定义

### 冷启动阶段（无反馈）
先用规则过滤：相关性、信息密度、新鲜度、商业/风险信号、噪音惩罚。分数低于阈值不进简报。

### 有反馈后（持续迭代）

飞书卡片或命令行提交 👍/👎 后，学习环节会：

1. 将每条反馈以 JSON 一行追加到 `data/feedback.jsonl`（事件日志，便于审计与重算）。  
2. 全量重算聚合权重，覆盖写入 `data/preferences.json`（排序时直接读取的偏好表）。

## 4. 简报格式

设计原则：先给跨赛道核心洞察，再按固定赛道展开；每条目固定字段、可追溯原文；末尾附本日统计与反馈说明。

简报样例（内容为示意缩写）：

```markdown
# 每日投资信息简报 | 2026-04-22

## 📈 今日核心洞察 (AI生成)
- 跨赛道要点一（概括当日入选内容的共同主线）。
- 跨赛道要点二。
- 跨赛道要点三。

## 🤖 AI 领域 (2/2条)

**标题：** 示例标题
**内容总结：** 若干句：事实摘要 + 与投融资相关的解读。
**投资信号：** 利好 / 风险 / 待观察 等标签 + 一句理由。
**原文链接：** [🔗](https://www.youtube.com/watch?v=xxxx) YouTube · @频道名
**用户反馈：** 若已配置飞书推送，请在群内简报卡片中展开对应赛道，在条目下点击 👍 / 👎。

**标题：** 第二条示例
```

这样设计的原因：`今日核心洞察` 先给当日横截面结论；各赛道标题中带 `(已选条数/本赛道候选条数)`，便于感知覆盖面；每条固定「标题 / 内容总结 / 投资信号 / 原文链接 / 用户反馈」五段，便于飞书卡片与本地 Markdown 同源、跨日对比；`本日数据统计` 与末行说明把规模与反馈闭环写清。
